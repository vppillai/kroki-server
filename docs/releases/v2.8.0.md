# DocCode v2.8.0

DocCode is now production-deployable for the public internet. The closed-network
default experience is unchanged — no `.env` changes are required for existing
self-hosters.

---

## Highlights

**Production-deployable.** A new end-to-end production guide
([docs/production-deployment.md](../production-deployment.md)) covers threat model,
TLS, hardening, sizing, and AI cost management — everything needed to safely expose
DocCode to the internet.

**Real TLS (Let's Encrypt).** `TLS_MODE=acme` issues and auto-renews Let's Encrypt
certificates via a certbot sidecar. Set `HOSTNAME`, `HTTPS_PORT=443`, `TLS_MODE=acme`,
and `ACME_EMAIL`; the setup script handles the rest. See [docs/acme-tls.md](../acme-tls.md).

**Always-on render cache.** An nginx proxy cache now absorbs repeat and burst traffic
on GET render routes. Deterministic renders (`/<type>/<format>/<encoded>`) are cached;
POST and `/api/` routes are never cached. The cache is always enabled by default
(`RENDER_CACHE_ENABLED=true`); set `RENDER_CACHE_ENABLED=false` to opt out.

**Deployment profiles / footprint trimming.** `DEPLOY_PROFILE=public` activates
internet-safe nginx limits (10 r/s, 1 MB body cap, 30 s timeouts). `COMPOSE_PROFILES`
and `DISABLED_DIAGRAM_TYPES` let you run only core + mermaid, saving ~4.7 GB of image
pulls and ~0.5–1 GB of idle RAM.

**Three-state AI mode + free-tier allowlist.** The AI posture is now explicitly
relay / byok / off. `AI_MODEL_ALLOWLIST="*:free"` restricts the relay to free-tier
models on OpenRouter, making a zero-cost public AI relay practical.

**BYOK privacy guarantee with verification doc.** In BYOK mode the API key never
touches the server. This is structurally enforced in code, locked by regression tests,
and now documented with DevTools verification steps in
[docs/byok-privacy.md](../byok-privacy.md).

**CSP + vendored dependencies.** Content-Security-Policy headers are active; all
previously CDN-fetched JS/CSS is now vendored or import-mapped. The
`IMPORTMAP_SHA256` integrity value and its regression test lock any future change.

**Gunicorn app server.** The demosite container runs gunicorn (gthread workers) —
not the Flask dev server. Tune with `GUNICORN_WORKERS`, `GUNICORN_THREADS`, and
related vars. The dev server (`python demoSite/server.py`) still works for bare
local development outside Docker.

## Security

**CSP headers active.** A strict `Content-Security-Policy` header is now emitted
on all responses, protecting against XSS and data-exfiltration.

**esm.sh removed.** All previously CDN-fetched ES modules are now vendored locally.
No external script sources remain.

**KROKI_SAFE_MODE now explicitly pinned** — was already the image default (secure).
The `.env.example` and `docker-compose.yml` now make this explicit so the setting
is visible, documented, and overridable. `secure` mode blocks PlantUML
`!include`/`!includeurl` and remote image fetches (SSRF/LFI guard).

**KROKI_BODY_LIMIT.** A body-size cap at the Kroki core layer provides
defense-in-depth behind the nginx body limit.

**Security findings addressed (H1–H7 summary):**
- H1: Structural guard prevents BYOK key leaking into proxy requests
- H2: XSS payload in AI model field is entity-escaped before render
- H3: CSP header blocks inline script execution and unauthorized origins
- H4: vendored JS eliminates CDN supply-chain risk
- H5: HSTS emitted in acme mode; max-age=86400 with upgrade path to 31536000
- H6: Server pops client-supplied key fields before any logging code path
- H7: `AI_ACCESS_TOKEN` bearer-token gate for the relay endpoint

## Behavior changes

**Always-on render cache + X-Cache-Status.** The nginx proxy cache is enabled by
default. Cached GET render responses include `X-Cache-Status: HIT` or `MISS`.
To disable: `RENDER_CACHE_ENABLED=false`.

**Key-less relay now yields byok mode instead of request-time 400.** Previously,
setting `AI_ENABLED=true` with no `AI_PROXY_API_KEY` caused a 400 at call time.
Now the server detects the empty key at startup and advertises `byok` mode — the
assistant UI stays visible and guides users to bring their own key.

**Default model gpt-4o → gpt-4o-mini.** The default `AI_MODEL` is now
`openai/gpt-4o-mini` (cost-conscious default). Override with any model your proxy
supports.

**Docker Compose v2 >= 2.24 now required.** Required for `--profile '*'` teardown
and `deploy.resources.limits` support. Verify with `docker compose version`.
This version has been available since February 2024 and is production-verified
on the reference deployment.

**`ai.enabled` semantics tightened.** The `/api/config` response `ai.enabled`
field now reflects the computed mode (`relay`/`byok`/`off`), not just the raw
`AI_ENABLED` env var.

**HSTS in acme mode.** `TLS_MODE=acme` now emits
`Strict-Transport-Security: max-age=86400`. Browsers that receive this header will
refuse HTTP connections to your hostname for up to 24 hours after any rollback.
The 86400 (1 day) initial value is deliberate rollback safety.

## Upgrade notes

**Closed-network self-hosters:** no `.env` changes are required. All new variables
default to behavior-preserving values. `KROKI_SAFE_MODE=secure` was already the
image default; the explicit pin is a no-op for existing deployments.

**Docker Compose version floor:** ensure `docker compose version` reports >= 2.24.0
before upgrading.

**Cache flush after core image upgrade:** if you upgrade the Kroki core image,
flush the render cache to prevent stale renders:
```bash
docker compose down
docker volume rm $(docker volume ls -q | grep nginx_cache)
./setup-kroki-server.sh start
```

**Public deployment guide:** see [docs/production-deployment.md](../production-deployment.md)
for the full hardening checklist before exposing this instance to the internet.
